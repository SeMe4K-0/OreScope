# Сценарий видео-демо (≤ 5 минут)

**0:00–0:30. Проблема.** Кадр панорамы 300 Мп. Закадрово: «Геолог тратит часы на ручную
разметку одного шлифа, оценки субъективны. OreScope делает это за две минуты и по одному
правилу».

**0:30–1:00. Запуск.** `start.bat` → браузер `localhost:7861`. Показать пустой дашборд,
легенду цветов в сайдбаре.

**1:00–2:00. Пакетная загрузка.** Перетащить 3–4 файла (рядовая, труднообогатимая,
оталькованная, панорама), указать мкм/px. Показать очередь: статусы сами меняются
«В очереди → Обработка → Готово».

**2:00–3:20. Разбор результата.** Открыть оталькованную:
- слой «Маска»: зум колёсиком в синие зоны талька, красные тонкие срастания;
- слой «Уверенность»: спорные участки;
- таблица: доли в % и мм²; заключение «Руда классифицирована как оталькованная:
  содержание талька — …»;
- предупреждение при пограничном тальке (если попадётся) — флаг «Проверка».

**3:20–4:00. Панорама.** Открыть готовую панораму 20k×14k: маска без швов, время ~2 мин
на 300 Мп (норматив ТЗ — 5 мин на 100 Мп). Экспертный вердикт «Подтвердить».

**4:00–4:40. Экспорт.** PDF-отчёт (открыть), CSV, GeoJSON в QGIS/geojson.io, JSON-лог
параметров (воспроизводимость).

**4:40–5:00. Финал.** Кадр с метриками: F1 срастаний 0.91, панорама за 124 с. «Код,
Docker и руководство пользователя — в репозитории».

Запись: OBS / Xbox Game Bar (Win+G), 1080p. Микрофон опционально — можно субтитрами.
