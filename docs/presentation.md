# OreScope — черновик презентации (10–12 слайдов)

## 1. Титул
**OreScope** — автоматическая классификация руд по панорамным OM-изображениям шлифов.
Команда, хакатон Норникель, июль 2026.

## 2. Проблема
- Ручная оценка шлифа геологом: **часы на образец**, панорамы до гигапикселей.
- Субъективность: согласие даже опытных экспертов по пиксельной разметке — Dice ≈ 0.77
  (IEEE JSTARS 2025, SulfideNet) — разные лаборатории получают разные ответы.
- «Преобладание» срастаний на глаз ≠ проценты для технологических карт.

## 3. Задача (экспертное правило)
Тальк > 10 % → **оталькованная** · иначе преобладают обычные срастания → **рядовая**,
тонкие → **труднообогатимая**. Требования: маска (зелёный/красный/синий), таблица метрик,
заключение, ≤ 5 мин на панораму 10k×10k, MAE талька ≤ 3 п.п., F1 срастаний ≥ 0.90.

## 4. Данные (что реально пришло)
- 14 панорам 13k×9.5k … 20.6k×14.6k (до 300 Мп), без разметки;
- ч1: 178 фото 2272×1704, 3 сорта; тальк размечен **синей линией от руки** (42 пары,
  контуры местами не замкнуты!);
- ч2: ~1000 фото 4160×2768 с меткой сорта;
- три разных масштаба съёмки и жёсткий доменный сдвиг: обучающие фото жёлто-оливковые,
  панорамы тёмные нейтральные.

## 5. Решение (пайплайн)
Схема: предобработка (валидная маска, освещение, царапины) → сульфиды (адаптивный порог)
→ срастания (CNN на weak labels + масштабо-инвариантная морфология) → тальк (U-Net,
тайлы с косинусным смешиванием, talc &= ~sulfide) → правило + пограничные случаи → отчёт.

## 6. Разметка талька: из линий — в маски
- Автосшивание разрывов: мостики между концами штрихов, дотяжка до рамки, замыкание хорд;
- contact-sheet всех 42 пар + точечные ручные правки (overrides.json);
- негативные контроли в обучении (методология SulfideNet) — борьба с ложным тальком.

## 7. Срастания: интерпретируемость + точность
- Морфология по зёрнам: степень замещения, локальная плотность — инварианты масштаба
  (работают на 5x, 10x и панорамах одинаково) → потолок F1 0.77;
- CNN (resnet18) на weak labels ~1050 фото → **image-level macro-F1 0.91** (цель ≥ 0.90);
- итог: CNN классифицирует, морфология объясняет (степень замещения в отчёте).

## 8. Демо дашборда (скриншоты)
Партия → очередь → маска/уверенность с зумом → таблица (%, мм²) → заключение →
PDF/CSV/GeoJSON. Экспертный вердикт + сбор правок для дообучения.

## 9. Метрики (hold-out)
- Срастания: macro-F1 **0.91**;
- Сорт руды end-to-end: accuracy ~0.76 (узкое место — тальк, честно показываем);
- Панорама 300 Мп: **124 с** (норматив ТЗ 5 мин на 100 Мп — 12-кратный запас);
- Негативные контроли: ложный тальк в среднем ~2.5 % (после дообучения с негативами — обновить).

## 10. Соответствие ТЗ (чек-лист)

маска-оверлей ✓ · таблица + CSV ✓ · PDF ✓ · текстовое заключение ✓ · веб-интерфейс ✓ ·

## 11. Ограничения и развитие
Гигапиксели → потоковое чтение (pyvips); точность талька упирается в качество линий
разметки → active learning из data/corrections; SAM-ассистированная доразметка.
