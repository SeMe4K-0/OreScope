# План качественного обучения моделей OreScope (v2)

Дата: 2026-07-04. План опирается на фактический аудит `ore_data/` и текущее состояние
репозитория (модели v1 обучены, метрики в `outputs/evaluation.json`).

## 1. Аудит данных (факты)

| Источник | Кол-во | Разрешение | Цветовой тракт | Разметка |
|---|---|---|---|---|
| ч1 / Оталькованные | 42 + 42 пары | 2272×1704 | жёлто-зелёный | синие контуры талька (местами **разомкнутые**) |
| ч1 / Рядовые | 68 | 2272×1704 | жёлто-зелёный | только метка класса |
| ч1 / Труднообогатимые | 68 | 2272×1704 | жёлто-зелёный | только метка класса |
| ч2 / оталькованные | 87 | 6240×4160, 4160×2768 | тёмный нейтральный | **только метка класса, масок талька нет** |
| ч2 / рядовые | 497 | — " — | — " — | метка класса; есть `.bmp` |
| ч2 / тонкие | 418 | — " — | — " — | метка класса; есть `.bmp` |
| Панорамы | 14 | 148–573 Мп (16.jpg: 27025×21227) | тёмный, как ч2 | **нет разметки** — целевой домен деплоя |

Критические особенности, влияющие на обучение:

1. **Доменный сдвиг ч1 ↔ ч2/панорамы**: ч1 — жёлто-зелёный тракт старой камеры;
   ч2 и панорамы — тёмные нейтральные снимки. Вся пиксельная разметка талька — только в ч1.
2. **Масштаб не унифицирован**: в именах ч1 есть увеличение («5x», «10x», «20x»);
   в ч2 в правый нижний угол **вожжена белая плашка со шкалой «300 мкм»**.
   Плашка — источник двух багов: белый текст попадает в маску сульфидов,
   а CNN может выучить её как shortcut-признак домена/класса.
3. **Риск утечки по пробам**: имена ч1 вида `2550374-2`, `2550375-1` — это
   `<проба>-<шлиф/поле>`. Один шлиф даёт несколько фото. Сплит train/val обязан
   идти по **ID пробы**, не по файлам. В ч2 имена не несут ID — принимаем файл=образец,
   риск фиксируем в допущениях.
4. **Морфология классов** (визуальная проверка): рядовые — массивные светлые сульфиды
   с минимальным замещением; тонкие — «скелетные»/решётчатые срастания и (в ч2) мелкая
   сульфидная вкрапленность внутри серых агрегатов; тальк — тёмная рассеянная фаза
   в нерудной матрице, границы визуально нечёткие (эксперт вёл контур «примерно»).
5. `.bmp` в ч2 сейчас игнорируются загрузчиком (фильтр только jpg/jpeg/png).

## 3. Этап 0 — фундамент данных (без него обучение «в мусор»)

0.1. **Реестр данных** `data/registry.parquet`: путь, домен (ч1/ч2/пано), класс, проба-ID
     (парсинг префикса имени ч1), увеличение (из имени: 5x/10x/20x), размер, µm/px.

0.2. **Физический масштаб**: вычислить µm/px для ч2 по шкале «300 мкм» (детект плашки —
     фиксированный ROI правого нижнего угла + подтверждение по шаблону); для ч1 —
     из увеличения объектива (таблица калибровок микроскопа, уточнить у заказчика;
     до уточнения — относительная нормировка ч1 5x/10x/20x между собой).

0.3. **Маска артефактов**: плашка шкалы ч2 → в `valid_mask` (исключить из сульфидов,
     из кропов CNN и из знаменателя площадей). Это правка `src/preprocessing.py`,
     влияет и на инференс, и на подготовку датасетов.

0.4. **Сплиты — один раз и навсегда**: `data/splits/{talc,grain,e2e}.json`.
     GroupKFold по пробе-ID (ч1) и по файлу (ч2); стратификация по классу и домену.
     Все скрипты обучения/оценки читают сплиты из файлов, а не генерируют сами.

0.5. **`.bmp` поддержать** в загрузчике (или конвертировать в PNG при индексации).

0.6. **Цветовая нормализация** как ступень предобработки (а не только аугментация):
     gray-world / Reinhard-перенос к референсному профилю + CLAHE по L-каналу.
     Кандидат-решение «обучать в масштабо- и цвето-нормализованном пространстве»
     проверяется абляцией на этапе 2.

Выход этапа: воспроизводимый датасет-реестр + фиксированные сплиты + µm/px.

## 4. Этап 1 — качество разметки талька (главный рычаг для MAE)

1.1. **Ревизия 42 масок**: перегенерировать `make_talc_masks.py`, отсмотреть
     contact-sheet все 42 пары; правки — через `data/talc_masks_overrides.json`.
     Отдельно проверить замыкание разомкнутых контуров (наблюдалось на
     `2550374-2 10х`): правило замыкания — по границе кадра и кратчайшей хорде,
     спорные — вручную.

1.2. **Доразметка ч2** (закрыть доменную дыру): SAM/интерактивная разметка
     15–25 фото из «ч2/оталькованные» силами эксперта (грубые полигоны достаточно —
     метрика площадная). Приоритет отбора — active learning: фото, где текущая
     модель максимально неуверенна (энтропия TTA) либо предсказывает долю near-10%.

1.3. **Слабая супервизия остального ч2-отальк** (62–72 фото): pseudo-labels от
     лучшей модели, фильтр по уверенности (p>0.8 / p<0.2, серая зона игнорируется
     в лоссе), только после того как 1.2 даст честный ч2-валидационный срез.

1.4. **Негативы**: расширить с 40 до ~120 (ч1+ч2 рядовые/тонкие, сбалансировано
     по доменам), в train only. Цель — прижать «ложную оталькованность» (сейчас 6.7%>10%).

Выход этапа: ≥57 честных масок в двух доменах + пул псевдоразметки + негативы.

## 5. Этап 2 — U-Net талька v2

2.1. **Протокол оценки прежде обучения**: 5-fold CV по пробам (42+новые), метрики
     на **полных изображениях** (не тайлах): Dice, MAE площади в п.п., калибровочная
     кривая pred-vs-GT доли. Отдельный **замороженный holdout**: 8 ч1 + 8 ч2 фото,
     в CV не участвуют. Оценка n=8 из v1 статистически пуста — решения принимаем
     только по CV+holdout.

2.2. **Абляции** (по одной осмысленной оси за раз, всё логируется в
     `outputs/experiments.csv`: конфиг-хеш, сплит-хеш, метрики):
     - вход: RGB-нормализованный vs градации серого+CLAHE (снятие цветового домена);
     - масштаб: тайлы в едином µm/px (ресайз по реестру) vs текущий пиксельный;
     - энкодер: resnet34 (базлайн) → efficientnet-b3 / convnext-tiny; декодер U-Net vs DeepLabV3+;
     - лосс: Dice+CE (базлайн) → + boundary loss; для MAE — **прямая калибровка площади**
       (изотоническая/линейная регрессия pred→GT доли на CV-предсказаниях) как
       постпроцессинг — дёшево и бьёт прямо в целевую метрику ТЗ;
     - тайл 512 → 768/1024 (контекст рассеянной фазы).

2.3. **Аугментации**: сохранить доменную симуляцию (затемнение/десатурация, царапины),
     добавить смешивание доменов после нормализации цвета; НЕ вводить сильные цветовые
     сдвиги, ломающие диагностичность фаз.

2.4. **Self-training на целевом домене**: псевдоразметка панорам (тайлы) и ч2-отальк
     (из 1.3), 1–2 раунда, только высокоуверенные пиксели, с контролем на ч2-holdout —
     если метрика ч2 не растёт, раунд откатывается.

2.5. **Калибровка вероятностей** (temperature scaling на val) — тогда карта уверенности
     и порог `talc_prob_thresh` осмысленны; порог подбирается свипом **по MAE площади
     на CV**, отдельно проверяется стабильность по доменам.

Критерий выхода: MAE ≤ 3 п.п. на замороженном holdout (оба домена), ложный тальк
>10% на негативах ≈ 0, Dice ≥ 0.75.

## 6. Этап 3 — классификатор срастаний v2 (weak labels → надёжные 0.90+)

3.1. **Пересобрать кропы** с учётом этапа 0: маска плашки, µm/px-нормализация
     (кроп фиксированного физического размера, например 1×1 мм, затем 224/384px),
     фильтр «есть сульфиды» — не глобальный Otsu (на тёмных ч2 он ловит магнетит),
     а порог по перцентилю яркости в валидной зоне + минимальная площадь зерна в мкм².

3.2. **Агрегация вместо «метка кропа = метка фото»**: MIL-подход — attention-pooling
     или top-k среднее по кропам изображения; лосс на уровне изображения. Это убирает
     основной шум слабых меток (на «тонком» фото бывают массивные зёрна и наоборот).

3.3. **Чистка меток**: после первого прогона — confident-learning (ранжирование
     кропов по расхождению с меткой), просмотр топ-100 подозрительных, исключение/перевзвешивание.

3.4. **Баланс доменов и классов в батче** (ч1 всего 136 фото против 915 в ч2 —
     семплер по стратам домен×класс), доменная аугментация симметричная
     (ч1→«тёмный», ч2→«жёлтый»).

3.5. **Оценка**: GroupKFold по пробам, macro-F1 image-level; обязательный срез
     **по доменам отдельно** (ч1-F1 и ч2-F1) и leave-one-domain-out тест
     (train ч2 → test ч1 и наоборот) как индикатор переносимости на панорамы.

3.6. Морфологическая ветка (замещение/плотность) остаётся как интерпретируемый
     fallback и источник признаков; калибровка порогов — только на train-фолдах.

Критерий выхода: macro-F1 ≥ 0.90 в каждом домене (не только в среднем), LODO-F1 ≥ 0.80.

## 7. Этап 4 — сквозная калибровка правила и end-to-end оценка

4.1. Собрать **e2e-валидацию**: все ~1180 фото с метками классов через полный пайплайн
     (в v1 — 90 фото; считать все, это дёшево на масштабированных копиях).
     Confusion-анализ по доменам.

4.2. Свип порогов правила (`talc_percent` фиксирован ТЗ = 10%, но
     `talc_prob_thresh`, `min_sulfide_percent`, `near_tie_percent`) — по train-фолдам,
     отчёт на holdout. Ошибки «рядовая→оталькованная» (5/27 в v1) должны уйти за счёт
     негативов (1.4) и калибровки (2.5).

4.3. **Пограничные случаи** — отдельный мини-набор: тальк 8–12%, паритет срастаний,
     пустая порода; проверка, что система выставляет флаг «требуется проверка»,
     а не молча ошибается.

Критерий выхода: e2e accuracy ≥ 0.90 и macro-F1 ≥ 0.90 на holdout по фото;
все пограничные кейсы либо верны, либо помечены флагом.

## 8. Этап 5 — панорамы (целевой домен) и цикл дообучения

5.1. **Панорамный тест-сет**: нарезать по 3–5 тайлов 4096px из каждой из 14 панорам,
     эксперт даёт быструю оценку (класс тайла, примерная доля талька «на глаз», 30–60 сек/тайл).
     Это единственный способ узнать реальное качество в домене деплоя.

5.2. Прогнать все 14 панорам, зафиксировать время/память (цель ТЗ: 100 Мп ≤ 5 мин;
     проверить худший случай 573 Мп — при необходимости включить `downscale_max_side`
     для талька, сульфиды считать в полном разрешении).

5.3. **Active learning замкнуть**: правки эксперта из дашборда (`data/corrections/`)
     → `train_talc.py --resume` / дообучение CNN; регламент: раунд дообучения после
     каждых ~20 правок, с прогоном полного eval-харнесса до/после.

## 9. Сквозные требования к процессу

- **Воспроизводимость**: seed, конфиг-хеш, сплит-хеш, git-SHA и хеш датасета в каждом
  запуске (`outputs/experiments.csv` + JSON-лог); модели версионируются
  (`talc_v2_YYYYMMDD.pt`), `evaluation.json` — по версиям.
- **Один eval-харнесс** (`scripts/evaluate.py`) — единственный источник метрик;
  никакие «метрики из обучения» в отчёты не попадают.
- **Бюджет**: этапы 0–1 ~2–3 дня (в т.ч. время эксперта на доразметку),
  этап 2 ~2–3 дня GPU-итераций, этапы 3–4 ~2 дня, этап 5 ~1–2 дня + цикл.
- **Стоп-правило**: если после этапа 2 MAE талька на ч2-holdout > 5 п.п. —
  приоритет смещается на доразметку ч2 (ещё +15–20 масок), а не на архитектуры.

## 10. Риски и допущения

| Риск | Митигация |
|---|---|
| µm/px ч1 неизвестен точно (нет таблицы калибровки) | запросить у заказчика; до тех пор — относительная нормировка и масштабные аугментации |
| В ч2 нельзя сгруппировать фото по пробам (имена не несут ID) | допущение «файл = образец» зафиксировано; контроль переобучения через LODO ч1↔ч2 |
| Маски из контуров систематически смещены (эксперт вёл «примерно») | площадная калибровка pred→GT (2.2) компенсирует систематику; спорные маски — ревизия 1.1 |
| ±3 п.п. MAE может быть недостижимо на рассеянной фазе с нечёткой границей | measure first: если после этапов 1–2 MAE 3–5 п.п. — согласовать с заказчиком метрику «MAE в зоне решения 5–15% талька», где ошибка реально влияет на класс |
| Псевдоразметка закрепляет ошибки модели | серые зоны вне лосса, откат раунда по ч2-holdout (2.4) |
| Панорамы могут отличаться и от ч2 (стыки, виньетирование) | панорамный тест-сет (5.1) до любых выводов; косинусное смешивание тайлов уже в пайплайне |
